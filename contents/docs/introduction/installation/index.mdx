Para instalar e editar o Documents, você precisa ter vários pré-requisitos em vigor. Aqui está uma lista de todos os pré-requisitos essenciais
para configurar e trabalhar neste projeto.

## Pré-requisitos

<Step>
<StepItem title="Requisitos do Sistema">

- **Node.js:** Certifique-se de ter o Node.js instalado. O projeto usa `npm`, que vem com o Node.js. Instale o Node.js daqui.
- **NPM ou Yarn:** Como o projeto usa comandos `npm`, certifique-se de que o npm está instalado com o Node.js. Alternativamente, você pode usar Yarn se preferir.

Para verificar a instalação, execute:

```bash
node -v
npm -v
```

</StepItem>
<StepItem title="Git">
Você precisa ter o Git instalado para clonar o repositório. Baixe o Git [aqui](https://git-scm.com/).

```bash
git --version
```

</StepItem>
<StepItem title="Editor de Código">
Você precisará de um editor de código para trabalhar no projeto. Opções comuns incluem:

- [Visual Studio Code](https://code.visualstudio.com/)
- [WebStorm](https://www.jetbrains.com/webstorm/)
- Qualquer outro editor de texto de sua preferência.

</StepItem>
<StepItem title="Conhecimento Necessário">
- **Next.js:** Como este é um projeto Next.js, você deve estar familiarizado com os conceitos básicos do Next.js.
- **React:** O projeto usa React, então entender a estrutura de componentes e hooks do React será útil.
- **TypeScript:** O projeto é escrito em TypeScript, então você deve ter um entendimento básico dele.
- **Tailwind:** O projeto usa Tailwind CSS para estilização.
- **MDX:** Markdown + JSX (MDX) é usado para escrever a documentação.

</StepItem>
</Step>

## Instalação

<Step>
<StepItem title="Clonar o Repositório">
Primeiro, clone o repositório para sua máquina local usando Git:

```bash
git clone https://github.com/rubixvi/rubix-documents.git
```

Mude para o diretório do projeto:

```bash
cd rubix-documents
```

</StepItem>
<StepItem title="Instalar Dependências">
Use npm para instalar todas as dependências necessárias:

```bash
npm install
```

Isso instalará todas as dependências listadas no arquivo package.json, incluindo:

- Next.js
- React
- TypeScript
- Tailwind CSS
- Componentes MDX
- Outras dependências como ESLint, PostCSS, Husky, etc.

</StepItem>

<StepItem title="Executando o Servidor de Desenvolvimento">
Inicie o servidor de desenvolvimento executando:

```bash
npm run dev
```

Isso iniciará o servidor local, e você poderá abrir seu navegador em:

```arduino
http://localhost:3000
```

</StepItem>

<StepItem title="Editando o Projeto">
  - **Arquivos MDX:** O conteúdo da documentação está armazenado em `contents/docs`.
  - **Componentes:** Você pode editar componentes React no diretório `components/`.
  - **Tailwind:** As estilizações do Tailwind são gerenciadas em `tailwind.config.ts` e `app/globals.css`.

Para fazer alterações no projeto, abra os arquivos em seu editor e modifique-os conforme necessário.

<Note title="Obrigatório" type="danger">
  A funcionalidade de busca do projeto depende da automação do Husky para construir
  `search-data/documents.json`. Certifique-se de realizar um commit git para gerar este
  arquivo.
</Note>

</StepItem>
<StepItem title="Construindo para Produção">
Depois de fazer suas alterações e quiser construir o projeto para produção:

```bash
npm run build
```

Isso gerará uma build pronta para produção do aplicativo.

</StepItem>
<StepItem title="Executando o Servidor de Produção">
Para executar o servidor de produção:

```bash
npm run start
```

O aplicativo de produção estará disponível em:

```arduino
http://localhost:3000
```

</StepItem>
</Step>

## Opcional

<Step>
<StepItem title="Implantando no Vercel">
Este projeto é otimizado para implantação no Vercel. Para implantar no Vercel:

- Acesse [vercel.com](https://vercel.com/) e faça login.
- Vincule seu repositório GitHub e implante o projeto.
- O Vercel cuidará do processo de build automaticamente para você.
</StepItem>
<StepItem title="Linting e Hooks de Pré-commit">
O projeto usa ESLint para linting e Husky para Git hooks. Essas ferramentas são configuradas automaticamente, mas você pode personalizá-las nos diretórios `.eslintrc.json` e `.husky`.
</StepItem>
</Step>

## Informações Importantes

A funcionalidade de busca do projeto depende da automação do Husky para construir `search-data/documents.json`. Certifique-se de realizar um commit git para gerar este arquivo.
